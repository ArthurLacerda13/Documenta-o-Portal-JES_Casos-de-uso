Visão de Casos de Usos

Desenvolver artefatos de análise de sistemas baseada na Visão de Casos de Uso.  Para tanto, deverá ser criada uma pasta chamada specs/casos de uso na raiz da pasta de projeto da equipe. Nela deverá conter os diagramas utilizados para análise de todo o sistema. Para tanto, lembre-se que a ferramenta é:
Diagrama geral de casos de usos (obrigatório)
Especificação de cada caso de uso (obrigatório, especificação textual ou história de usuário)
Na prática e, no mínimo, nesta primeira etapa, será gerado um diagrama de casos de uso (geral) contendo caso de uso para cada um dos requisitos (a depender da análise). O que é esperado:
Um diagrama de caso de uso geral contendo todos os atores e casos de uso com relacionamentos.
Colocar no repositório as imagens dos diagramas em PDF ou SVG, conforme orientação inicial. As especificações textuais poderão ser feitas em formato .pdf, .doc, .docx ou .md (markdown).
